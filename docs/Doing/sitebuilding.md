## False Starts

### WordPress

I have been endeavouring to create this web site for several years. When doing my MSc, I did create a WordPress site which exists to this day, but it was something that (a) I kept playing with themes of (never being satisfied with a theme), and (b) I struggled t= that every time I returned to create a post that WordPress had changed the writing tools.

### Quarto

For various reasons, I ended up using *R* to do the statistical analysis required for my MSc thesis. THis took me to the [Posit](https://posit.co/) tools, notably [RStudio](https://posit.co/products/open-source/rstudio). 

I was aware of the ability to use RMarkdown as a publishing tool, but cognitively getting my brain around using and publishing made it a step too far at a time when I was trying to learn so many things at the same time

And then [Quarto](https://quarto.org/) happened... if you haven't seen QUarto from Posit, then it is a well engineered and superbly supported solution, that fits seamlessly into RStudio and so felt very familiar and easy to use (but then RMarkdown was already engineered into RStudio, and so I kink myself that I probably missed a trick there)

!!! Question "So why isn't this site created using Quarto?"

Good question! The answer is two fold:

- Firstly, while the Quarto documentation is excellent as showing how to publish on Posit's own free web hosting, it felt very *off the beaten track* internet wise.
- Secondly, Quarto had simply too many options, for me it was back to that cognitive overload, and I seemed to spending way too much time trying to make options that should have worked, work, only to find that under Option A, facility X was not enabled.
- Thirdly, RStudio takes a bit of getting used to as a serious writing tool. Quarto is very flexible, and the extensive support covers ways to use Quarto withe other IDEs.
- Finally (*this is getting very Monty Python*), trying to publish Quarto to [GitHub Pages](https://docs.github.com/en/pages) (where you are now) looked quite difficult compared to Posit's won hosting service.

Let me be clear, for serious publishing work, I'd persevere with Quarto as it is just so powerful. Maybe another blog post?

### Zensical

And so we arrive at this site, created using [Zensical](https://zensical.org/). I heard the lead developer, [Martin Donath](https://zensical.org/about/team/) talking about his work on a podcast. Zensical is a major update to a HTML documentation tool.

So yes, I am still climbing my way up a steep learning curve, but I got in the ground floor when, compared to Quarto, Zensical is relatively simple. And more importantly, the publishing to GitHub Pages is pretty well documented, hence my arrivel here.

It hasn't been plain sailing, again maybe another blog post

### What Have Learned Along the Way
